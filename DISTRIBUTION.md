# SnapText — Distribution Strategy

## The Core Message

**One line:** "Stop copy-pasting into ChatGPT. Reply to clients in one tap."

**The pain (relatable to every Indian professional):**
> Har baar client ka message aata hai → ChatGPT open karo → context paste karo → "reply professionally" likho → result copy karo → wapas jao → paste karo. 6 steps, 2 minutes. 15 times a day.

**The solution:**
> Select text → Tap SnapText → Reply ready. 3 seconds.

### Post Format Rules

1. **Hook in first 2 lines.** No "I'm excited to announce..." garbage.
2. **Screen recordings > text posts** for demos. People need to SEE the flow.
3. **Real messages, real scenarios.** Blur names but use actual WhatsApp/email screenshots.
4. **Never say "AI keyboard."** Say "reply assistant" or "writing shortcut."
5. **Hinglish in posts is fine.** Your audience thinks in Hinglish. Write for them.
6. **CTA at end:** "Link in comments" (LinkedIn suppresses posts with links in body).

### How to Record Screen Recordings

**Phone:**
1. Open WhatsApp → find a real client message
2. Start screen recording (swipe down → screen record)
3. Long-press the message → select text → tap SnapText
4. Show the reply generating → tap Insert
5. Stop recording. Trim the boring parts.
6. Add captions in CapCut or InShot (free apps)

**Desktop:**
1. Use OBS or Windows Game Bar (Win+G)
2. Show the ChatGPT flow first (slow, many tabs)
3. Then show SnapText flow (fast, one overlay)
4. Side-by-side comparison is gold

## What NOT to Do

- Don't pay for ads before you have 50%+ D7 retention
- Don't post generic "we launched our app!" announcements
- Don't optimize for downloads — optimize for daily active users
- Don't build features to "attract users" — build features that make existing users come back
- Don't compare yourself to Gboard or ChatGPT in marketing — just show the workflow
- Don't ask people to "share with friends" — if the product is good, they will
